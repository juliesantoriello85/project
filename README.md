# QA Automation Portfolio

This repository is intended for recruiters who want to review my QA automation work.

It highlights how I design and organize automated end-to-end tests using:
- Cypress
- Gherkin / BDD scenarios
- reusable step definitions
- structured test data and selectors

The goal is to demonstrate my ability to translate business requirements into readable automated test scenarios, while keeping the test suite maintainable and scalable.

## What You Can Find Here

This repository contains practical automation examples focused on realistic user journeys and validation rules.

At the moment, the main example is:
- `Bank_Transfer/`: a Cypress + Gherkin test project covering a bank transfer flow, including happy paths, validation rules, and role-based access scenarios

## What This Demonstrates

Through these examples, you can evaluate my ability to:
- write business-readable Gherkin scenarios
- automate end-to-end flows with Cypress
- separate test logic, selectors, and data
- cover both positive and negative test cases
- structure QA assets in a clear and professional way

## Purpose

This repository is not only a technical exercise. It is also meant to show my approach to test automation:
- readability for both technical and non-technical stakeholders
- maintainability of the automation suite
- focus on realistic functional coverage
- attention to validation, edge cases, and user roles

Each folder may represent a different automation exercise or use case. Please refer to the local `README.md` files inside each project for more detailed context.
